    实验步骤

    • 利用HLS进行系统设计，可分为3个阶段：

      • 1.利用HLS生成IP核；

      • 2.创建Block Design；

      • 3.系统的软硬件联调。

    1. 利用HLS生成IP核

    1.1 创建Vivado HLS项目

    • 在桌面双击Vivado HLS快捷方式，或在开始菜单->Xilinx Design Tools->Vivado HLS打开Vivado HLS工具。点击“Quick Start”下方的“Create New Project”，或点击菜单栏中的File->New Project。然后，填写项目名称，并选择项目所在的路径，如图2-4所示。

    图2-4 创建Vivado HLS项目

    • 点击Next，进入源文件添加窗口。在Top Function一栏内填写“matrixmul”，作为顶层函数名，如图2-5所示。

    图2-5 填写Top Function名

    • 点击Next，进入Test Bench的测试源文件添加窗口。此时暂不添加，继续点击Next，进入项目配置窗口。点击Part Selection线框内部的“…”按钮，打开器件选择窗口，输入xc7z020clg400-1，然后点击“OK”按钮，如图2-6所示。

    图2-6 选择器件

    • 点击“Finish”按钮，完成项目的创建，进入主界面，如图2-7所示。

    图2-7 Vivado HLS主界面

    • Vivado HLS主界面主要包括6个部分：1）菜单栏和工具条。菜单栏和普通的IDE差不多，尤其接近Eclipse，这是因为Vivado HLS就是基于Eclipse开发的。工具栏包含了常用的功能按钮，如综合、仿真、打包IP核等。2）视图切换按钮，用于在Debug、综合和分析3个视图之间切换。3）项目管理窗口，用于管理项目源文件、测试源文件、IP核压缩包等文件。4）工作区：用于编辑源码和调试。5）控制台和报错信息窗口：用于查看报错信息和串口输出等。6）Outline窗口：用于显示项目相关的信息，包括顶层函数输入输出变量所对应的硬件模块的接口信号总线协议等。

    1.2 添加源文件

    • 将所提供的matrixmul.cpp、matrixmul.h和matrixmul_test.cpp三个源文件拷贝至工程目录下（D:\DLAlab1\matrixmul）。然后在项目管理窗口的Source图标处右键，选择Add File，选择刚才拷贝的除matrixmul_test.cpp之外的源文件，如图2-8所示。

    图2-8 添加源文件

    • 双击Source下的源文件。可编辑其代码，如图2-9所示。

    图2-9 查看源代码

    • 类似的，在Test Bench图标处右键添加matrixmul_test.cpp，如图2-10所示。

    图2-10 添加测试源文件

    1.3 C代码仿真/CSIM

    • 点击Project->Run C Simulation或工具条中的CSim按钮，如图2-11所示。

    图2-11 点击进行CSim

    • 点击CSim后，直接在弹出的窗口中点击“OK”按钮。此时，可在控制台窗口查看源码的编译过程及相应的测试结果，如图2-12所示。

    图2-12 查看CSim结果

    1.4 C代码调试

    • 如果CSim时不能通过测试，则需要对代码进行debug。Vivado HLS工具的调试方法和Eclipse基本一致。

    • 点击CSim按钮，在弹出的窗口中勾选Launch Debugger，如图2-13所示。

    图2-13 开始debug

    • 点击“OK”按钮，将自动进入跟Eclipse一样的Debug视图。可在代码行数的左侧空白处双击以添加断点，如图2-14所示。

    图2-14 添加断点

    • 点击工具条中的Step Into(F5)或Step Over(F6)按钮，开始debug。此时，可在右上方的Variable窗口查看变量的值，如图2-15所示。

    图2-15 debug时查看变量的值

    • debug完成后，点击图2-15左上角形如红色正方形的按钮，或用快捷键Ctrl+F2结束调试。此时，再点击图2-15右上角的Synthesis按钮，回到图2-7所示的主界面/综合界面。

    1.5 综合/Synthesis

    • 综合前，需要为matrixmul项目添加Directive，以指定顶层模块所对应的硬件模块的输入输出信号所使用的总线协议、综合时所使用的并行优化策略（如流水线）等。

    • 首先需要设置matrixmul模块的输入输出信号所使用的总线协议。

    • 在右侧的Directive窗口中，在顶层模块上右键，点击Insert Directive。然后在弹出的Directive Editor窗口中，将Directive设置为INTERFACE，并将mode设置为s_axilite，如图2-16所示。

    图2-16 为顶层模块添加Directive

    • 分别在参数A、B和out上右键，点击Insert Directive。然后在弹出的Directive Editor窗口中，将Directive设置为INTERFACE，将mode设置为m_axi，将depth设置为1024，并将offset设置为slave，如图2-17所示。

    图2-17 为顶层模块的参数添加Directive

    • 接着，设置HLS综合时的并行优化策略，此处以流水线优化为例。

    • 在Col上右键，点击Insert Directive。然后在弹出的Directive Editor窗口中，将Directive设置为PIPELINE，在Options II中填入数字1，如图2-18所示。

    图2-18 为Col标签添加Directive

    • 最终的Directive窗口如图2-19所示。

    图2-19 最终的Directive窗口

    • 所有的Directive都保存在了directives.tcl文件中，如图2-20所示。

    图2-20 directives.tcl文件

    • Directive设置完毕，接下来需要对C代码进行综合，以生成RTL电路。

    • 点击Project->Project Settings->Synthesis，点击Top Function右边的Browse，选择matrixmul作为Top Function，点击“OK”按钮，如图2-21所示。

    图2-21 选择Top Function

    • 点击菜单栏的Solution->Run C Synthesis->Active Solution，或点击工具栏中形如绿色三角形的综合按钮，如图2-22所示。

    图2-22 点击以进行综合

    • 综合完成后会自动显示综合报告。报告中包含了预估性能、预估资源情况、电路时延、模块接口等信息，如图2-23所示。

    图2-23 综合报告

    1.6 C/RTL协同仿真/C/RTL Co-Sim

    • 运行C/RTL Co-Sim时，会调用HDL仿真工具。

    • 点击Solution->Run C/RTL Cosimluation，或点击工具条中形如对号的按钮，如图2-24所示。

    图2-24 点击进行C/RTL联合仿真

    • 在弹出的窗口中选中仿真工具为Vivado Simulator，语言选中Verilog，Dump Trace选择all，如图2-25所示。

    图2-25 联合仿真设置

    • 仿真报告显示了仿真的结果以及时延等信息，如图2-26所示。

    图2-26 联合仿真报告

    • 还可点击工具条中的波形按钮以查看波形，此时将打开Vivado的仿真波形界面，如图2-27所示。

    图2-27 查看仿真波形

    1.7 导出RTL并打包成IP核/Export RTL

    • 点击菜单栏的Solution->Export RTL，或点击工具条中形如棕色箱子的按钮，如图2-28所示。

    图2-28 点击以导出RTL

    • 随后将弹出导出IP核的设置窗口，保持默认即可，点击“OK”按钮。

    • IP核导出成功后，可在主界面/综合界面左侧的工程目录Solution1->impl->ip中找到.zip格式的IP核，如图2-29所示。

    图2-29 查看导出的IP核

    2. 创建Block Design

    2.1 创建Vivado项目

    • 在桌面双击Vivado快捷方式，或在开始菜单->Xilinx Design Tools->Vivado打开Vivado工具。点击“Quick Start”下方的“Create Project”，或点击菜单栏中的File->Project->New。然后，填写项目名称，并选择项目所在的路径，如图2-30所示。

    图2-30 创建Vivado项目

    • 点击“Next”按钮，勾选“Do not specify sources at this time”，然后继续点击Next按钮。然后在搜索框中输入xc7z020clg400-1，如图2-31所示。

    图2-31 输入PYNQ-Z2的器件型号

    • 一直点击“Next”按钮，最后点击“Finish”按钮，完成Vivado工程的创建。

    2.2 添加IP核

    • 点击左侧Flow Navigator->Project Manager->Settings，在IP->Repository中添加前文生成IP核的路径，如图2-32所示。

    图2-32 添加IP核

    • 添加完成后，将弹出确认窗口。在该窗口中，可查看IP核，如图2-33所示。

    图2-33 添加成功后的窗口

    • 点击“OK”按钮，回到Vivado主界面。

    2.3 搭建Block Design电路

    • 点击IP Integrator->Create Block Design，并点击“OK”按钮以新建Block Design，如图2-34所示。

    图2-34 新建Block Design

    • 在右上方工作区中的Diagram内，点击加号添加ZYNQ IP核，然后点击Run Block Automation，如图2-35所示。

    图2-35 添加ZYNQ IP核

    • 随后将弹出一个窗口，保持默认设置，点击“OK”按钮即可。

    • 接下来，需要为ZYNQ的PS端添加AXI总线接口，以连接矩阵乘法IP核。

    • 双击ZYNQ IP核，为ZYNQ添加S_AXI_HP0总线接口，如图2-36所示。

    图2-36 为ZYNQ添加AXI总线接口

    • 点击Diagram中的加号，添加矩阵乘法IP核，如图2-37所示。

    图2-37 添加矩阵乘法IP核

    • 点击Run Connection Automation，并勾选所有复选框，随后点击“OK”按钮，如图2-38所示。

    图2-38 自动连接ZYNQ IP核与矩阵乘法IP核

    • 通常自动布线得到的电路图都较不美观，不利于观察和分析电路。可点击原形箭头进行重新布线，如图2-39所示。

    图2-39 重新布线

    • 点击菜单栏的保存按钮，或用快捷键Ctrl+S保存Block Design。

    2.4 生成比特流并导出Overlay

    • 在源文件管理窗口中的Block Design文件上右键，点击“Create HDL Wrapper”，如图2-40所示。

    图2-40 为Block Design添加HDL顶层文件

    • 随后将弹出“Create HDL Wrapper”窗口，保持默认设置，然后点击“OK”按钮即可。

    • 接下来，需要生成整个硬件设计的比特流文件。

    • 点击菜单栏中的绿色箭头，生成比特流文件，如图2-41所示。如果希望加快比特流的生成速率，可以将“Number of jobs”设置为最大值。

    图2-41 生成比特流

    • 耐心等待比特流生成完毕，然后点击菜单栏的File->Export->Export Block Design以导出Block Design，如图2-42所示。

    图2-42 导出Block Design的.tcl文件

    • 到Vivado项目的路径下，找到生成的.bit和导出的.tcl文件，如图2-43所示。

    图2-43 找到所生成的比特流文件和.tcl文件

    • 将这两个文件拷贝出来，并重命名为matrixmul.bit和matrixmul.tcl备用。

    3. 系统联调

    • 将PYNQ-Z2和PC连接好之后，在PC的Web浏览器访问192.168.2.99:9090，以密码xilinx登录Jupyter Notebook。

    • 新建并打开matrixmul目录，点击“Upload”按钮，上传matrixmul.bit、matrixmul.tcl以及所提供的matrixmul.ipynb文件，如图2-44所示。

    图2-44 在新建的matrixmul目录中上传矩阵乘法的Overlay

    • 点击文件右侧蓝色的“Upload”按钮，开始上传，如图2-45所示。

    图2-45 上传.bit和.tcl文件

    • 如果上传失败，请多尝试几次。

    • 打开matrixmul.ipynb，点击Cell->Runn All开始测试。如果所设计的IP核功能正确，最终将输出如图2-46所示的结果。

    图2-46 matrixmul.ipynb的测试运行结果
